re:Invent 2025 - Chalk Talk - Origin management for multi-region applications with Amazon CloudFront

Based on the available description, this session explains origin management with Amazon CloudFront to build secure, resilient content delivery systems. It covers automatic failover with CloudFront and Route 53, granular routing with CloudFront Functions, origin security (S3 Origin Access Control and VPC private origins), hybrid integration via AWS Direct Connect, and Origin Shield for cache optimization.

Watch on YouTube
Key points
  • Automatic failover using CloudFront combined with Route 53 for resilience
  • Granular origin routing and traffic control via CloudFront Functions
  • Secure origin access with S3 Origin Access Control (OAC) and VPC private origins
  • Hybrid on-premises integration through AWS Direct Connect
  • Deploy Origin Shield to optimize caching and reduce origin load
So what
  • Design for higher availability: combine CloudFront and Route 53 failover to keep content delivered during origin outages.
  • Improve security posture by using S3 OAC and VPC private origins to limit direct origin exposure.
  • Support hybrid architectures by connecting on-prem infrastructure via AWS Direct Connect for consistent delivery.
  • Reduce origin stress and backend costs by using Origin Shield to centralize and optimize cache hits.
  • Use CloudFront Functions for fine-grained routing logic to direct traffic based on real-time rules.
